> 本文完整记录华为盘古大模型从2016年战略预判到2026年全面开源的十年历程，还原这位"全球AI先驱"的兴衰与重生。

## 导言：从"无人知晓"到"生态旗手"

2026年6月12日，华为开发者大会（HDC 2026）的舞台上，华为常务董事、终端BG董事长余承东说了一段意味深长的话：

> "在全世界都不知道大模型为何物的时候，华为就发布了盘古大模型，可以说是这个行业绝对的全球先驱者。后来因为各种各样的原因，没做好，不应该。"

紧接着，他又掷地有声地放话：

> "在我余承东的字典里，没有第二，只有第一。我们将带领团队一路赶超，从中国第一，走向将来的世界第一。"

这不仅是余承东个人的重新归位，更标志着华为大模型战略的全新起点。从2016年首次提出"智慧手机"概念，到2017年全球首款集成NPU的手机芯片横空出世；从2019年悄然登顶中文NLP榜单却未声张，到2021年盘古1.0震惊世界；从2023年盘古气象大模型登上《Nature》正刊，到2025年的首次开源，再到2026年openPangu 2.0的全面开源——华为盘古大模型的十年，是一部从硬件预研到全栈自主、从行业深耕到生态开放的宏大叙事。

本文将完整呈现华为盘古大模型从无到有、从有到优、从优到开放的每一个里程碑节点，包括各代版本的详细参数、各项"第一"与荣誉、全栈自主的技术突破、开源战略的演变、以及余承东重掌帅印后对"世界第一"目标的坚定宣示。

## 第一章 先见之明：盘古之前，华为的AI棋局（2016—2019）

### 1.1 2016年乌镇：余承东提出"智慧手机"

2016年11月16日至18日，第三届世界互联网大会在乌镇召开。在这场汇聚全球科技巨头的盛会上，余承东发表了题为《人工智能，未来已来》的主题演讲，提出了一个超越时代的理念：**"智慧手机"**。

余承东在当时便明确指出：

> "我们当下所处的数字时代，最主要的特征就是信息的不断跃进。过去数十年的里程碑中，信息从PC本地化，到互联网共享，再到移动互联网的随时随地共享，那么接下来就是人工智能。未来人工智能最有效的载体，是人工智能+智能手机，即我们所说的智慧手机。"

在华为的战略蓝图中，智慧手机将从四个方向展开布局：

- **端（身体）**：手机全面复制人类感知能力，包括计算机视觉、智能语言、海量传感器和端侧人工智能，让手机具备"本地决策"的能力。
- **联接（语言）**：通过Hi-Link智慧家居平台和车联网技术，打通智慧手机与现实世界的交流障碍。
- **云（大脑）**：基于大数据和机器深度学习的云端智能，为用户提供认知与决策能力。
- **芯（算力）**：端侧AI必须从芯片架构层面进行根本性的设计。

余承东后来回顾这段历史时说：

> "当时华为内部有一支'小分队'在探索大模型技术，比外界认知的要早得多。在全中国、全世界都不知道大模型为何物的时候，我们就开始了部署。"

这正是他所谓"全球先驱"论断的核心底气和依据。

### 1.2 2017年IFA：全球首款AI手机芯片麒麟970

2017年9月2日，华为在德国柏林国际消费类电子产品展览会（IFA）上向世界宣告了一款革命性的产品——**全球首款移动端AI芯片麒麟970**。这是整个手机行业的分水岭时刻。

采用10nm工艺制程、集成55亿个晶体管的麒麟970，其最大的闪光点并非传统的CPU或GPU规格，而是**首次在SoC中集成了独立的神经网络处理单元NPU**，创造性设计了HiAI移动计算架构。

**麒麟970 NPU 关键性能指标：**

| 指标 | 数值 |
|------|------|
| 工艺制程 | 10nm |
| 晶体管数量 | 55亿 |
| NPU模块 | 寒武纪-1A |
| 性能（相比CPU） | 25倍 |
| 性能（相比GPU） | 6.25倍 |
| 能效比（相比CPU） | 50倍 |
| 能效比（相比GPU） | 6.25倍 |
| 图像识别速度 | 2005张/分钟 |

2017年10月，搭载麒麟970的华为Mate 10系列发布，余承东在发布会上将其称为"**全球第一款人工智能手机**"。

外界评价这段历史时曾说：

> "在大家还不知道AI是什么的时候，华为就已经把AI做进了手机。"

### 1.3 2018—2019：全栈AI地基的搭建与"暗中的冠军"

**2018年10月10日**，华为在上海召开的全联接大会上发布了基于"达芬奇架构"的两颗AI芯片：

- **昇腾910**：面向云端超大规模AI训练场景
- **昇腾310**：面向终端低功耗AI推理场景

与此同时，华为还推出了：
- CANN异构计算架构
- MindSpore全场景AI计算框架
- ModelArts一站式AI开发平台

从芯片到框架再到开发平台，一个从底层硬件到上层应用的全链路AI生态系统开始成型。

**2019年，盘古大模型在中文自然语言处理领域悄悄拿到了一个"世界冠军"。**

据权威的中文语言理解评测基准CLUE榜单显示，盘古NLP大模型在2019年以总成绩**83.046分**的绝对优势登顶榜首，在分类、阅读理解单项均排名第一，**一举刷新三项榜单世界历史纪录**，多项子任务得分接近人类理解水平85.61分。

然而，盘古的登顶没有新闻发布会，没有舆论宣传。因为在华为内部看来，这只是大模型基础研究的"预期成果"，还没有到将其作为独立产品对外发布的时机。

**2020年3月**，华为云首席科学家田奇正式加入并着手组建大模型研发团队。**11月，盘古项目在华为云内部正式立项**，进入了大规模"攻坚"阶段。

## 第二章 破晓：盘古大模型的诞生与高光（2021—2024）

### 2.1 盘古1.0（2021年4月）：全球最大的中文大模型正式登场

**2021年4月25日**，余承东在华为开发者大会（Cloud）上以华为云CEO的身份，正式发布了盘古系列超大规模预训练模型。这是中国大模型历史上的"开天辟地"时刻。

**盘古1.0 核心参数：**

| 模型 | 参数量 | 训练数据 | 成就 |
|------|--------|----------|------|
| 盘古NLP大模型 | 2000亿 | 40TB中文文本 | 全球最大中文语言模型，CLUE榜首83.046分 |
| 盘古CV大模型 | 30亿+ | ImageNet等 | 业界最大视觉预训练模型，小样本分类SOTA |

**盘古NLP大模型**是业界首个千亿参数的中文语言预训练模型，参数规模达到2000亿，超过了OpenAI当时的旗舰模型GPT-3（1750亿参数），是**当时全球最大的中文语言模型**。

**盘古CV大模型**是当时业界最大的视觉预训练模型，包含超过30亿参数，**首次兼顾了图像判别与生成能力**。在ImageNet数据集的小样本分类任务中达到了业界最高水平。

盘古大模型开创的工业化AI开发新模式——**"预训练+下游微调"**，在当时正切中行业痛点。盘古的口号是让AI开发由"作坊模式"转变为"工厂模式"。

### 2.2 盘古2.0（2022年4月）：从"全能"走向"分层"

2022年4月，盘古大模型2.0发布，其核心突破在于提出了**L0/L1/L2的"三层开发方案"**：

- **L0 基础层**：五大基础模型（NLP、CV、多模态、科学计算和预测模型）
- **L1 行业层**：针对政务、金融、制造等特定行业的"N"个行业模型
- **L2 场景层**：面向虚拟助手、台风路径预测等具体应用场景的"X"个模型

这种分层解耦架构使得客户既可以像使用"乐高积木"一样直接选用现成的行业解决方案，也可以在自己的私有数据上进行微调适配。

### 2.3 盘古3.0（2023年7月）："不作诗，只做事"

**2023年7月6日**，国际顶级学术期刊《Nature》正刊发表了华为云盘古气象大模型的研发成果。这篇论文是**中国科技公司首篇作为唯一署名单位发表的《Nature》正刊论文**。

**盘古气象大模型核心技术指标：**

| 指标 | 数值 |
|------|------|
| 预报速度 | 10秒完成全球7天预报 |
| 单次推理时间 | 1.4秒（V100显卡，24小时预报） |
| 计算速度提升 | 相比传统数值预报提升10000倍以上 |
| 功耗降低 | 达10000倍以上 |
| 水平空间分辨率 | 0.25°×0.25° |
| 时间分辨率 | 1小时 |
| 垂直高度层 | 13层 |

盘古气象大模型攻克了此前AI气象预报的两大技术难点：

1. 基于**3D Earth-Specific Transformer（3DEST）**处理复杂的不均匀3D气象数据
2. 使用层次化时域聚合策略，将预测特定时间气象状态的迭代次数压缩到最小

该模型在常用检验指标上优于全球最先进的欧洲中期天气预报中心（ECMWF）的确定性业务预报。它将ECMWF集成预报系统的预报时效提高了约**0.6天**，在热带气旋路径预报误差上较ECMWF降低了**25%**。

**盘古气象大模型获得的荣誉：**

- 《Nature》正刊发表（2023年7月6日）
- 《Science》杂志评为**2023年度十大突破之一**
- 入选国家自然科学基金委员会发布的"**2023年度中国科学十大进展**"，位列**第一**

**这是盘古大模型历史上的最高光时刻。**

### 2.4 盘古5.0（2024年6月）：万亿级全栈自主的里程碑

**2024年6月21日**，华为开发者大会首日，盘古大模型5.0发布。这是一次跨越式升级，在**全系列、多模态、强思维**三大方向上实现了质的飞跃。

**盘古5.0 三大突破方向：**

1. **全系列**：首次提供从十亿级到万亿级的全规格模型
2. **多模态**：支持高达10K超高分辨率图像理解
3. **强思维**：将思维链（Chain of Thought，CoT）与策略搜索技术深度结合

**盘古5.0最大突破**：2025年5月（资料追溯），盘古5.0**首次实现1.8万亿参数的全栈自主训练，完全基于昇腾910C芯片，未使用任何英伟达GPU**，成为全球首个全链路国产化的万亿级通用大模型。

### 2.5 盘古5.5（2025年6月）：MoE时代与首次开源

**2025年6月20日**，张平安正式发布盘古大模型5.5。

**盘古5.5 NLP模型关键参数：**

| 指标 | 数值 |
|------|------|
| 总参数量 | 7180亿 |
| 专家子网络数量 | 256个 |
| MoE架构 | 混合专家（Mixture-of-Experts） |
| 激活参数量 | 约390亿 |
| 推理效率提升 | 比前几代提升8倍 |

5.5还引入了**"快慢思考合一"机制**，根据任务复杂度动态切换推理模式：简单问题快速响应（快思考），复杂任务深度推演（慢思考）。

**2025年6月30日**，华为宣布**首次开源**，开源盘古7B稠密模型和72B MoE模型。这一步虽然仍停留在"开放权重"层面，但它标志着华为大模型战略的一个微妙转向——从封闭的行业工具，开始向开放的生态基石迈进。

## 第三章 破壁者：华为的算力分发与昇腾生态崛起

### 3.1 从"闭门造车"到"开放生态"

余承东在2026年的发布会上坦率地说明了华为真正的战略落脚点：

> "华为把昇腾算力大量支持了国内企业的需要，自己留的算力数量非常有限。"
>
> "美国企业有几十万卡、上百万卡等更高性能的算力资源，华为自己留的卡数量非常有限，还不能满足去训练几十万亿参数的大模型。"

这番话清晰地揭示了华为盘古大模型的核心战略价值：**盘古不是终点，而是证明昇腾"能行"的样板工程。**

### 3.2 盘古的"样本意义"：证明昇腾可训

**2025年4月**，华为盘古团队成功开发出完全基于昇腾算力训练的千亿级通用语言大模型——**Pangu Ultra**。

**Pangu Ultra 关键技术指标：**

| 指标 | 数值 |
|------|------|
| 总参数量 | 1350亿 |
| 架构层数 | 94层 |
| 训练集群 | 8192张昇腾NPU |
| 算力利用率（MFU） | 52% |
| 训练数据量 | 13.2万亿tokens |
| 训练稳定性 | 全流程无损失尖峰（loss突刺） |

整个训练过程**"英伟达含量为零"**。Pangu Ultra从数据清洗到预训练到强化学习，全流程完全基于华为昇腾NPU和昇思MindSpore AI框架。

### 3.3 昇腾生态的开花结果：DeepSeek、智谱、GLM、美团、Kimi的全面转向

盘古用Pangu Ultra验证之后，国内大模型厂商开始大规模从英伟达GPU生态转向昇腾NPU生态。

**DeepSeek**：2026年6月，基于昇腾910C的算力集群成功完成了**1.6万亿参数大模型DeepSeek-V4-Pro的全参数后训练**。训练模型迭代超过1500步，全程无迭代跳过或NaN异常。

**智谱AI（GLM）**：**2026年1月**，智谱联合华为开源新一代图像生成模型**GLM-Image**。这是**首个在国产芯片上完成全程训练的SOTA多模态模型**。在CVTG-2K复杂视觉文本生成榜单上，GLM-Image以0.9116的Word Accuracy成绩位列开源模型第一。

**美团LongCat**：**LongCat-2.0-Preview**是训练推理全程基于国产算力的万亿参数大模型，直接跑在国产算力之上，使用了**5万至6万张国产算力芯片**。这是全球目前唯一一个完全由国产卡训练的万亿大模型。

**月之暗面（Kimi）**：2025年11月，月之暗面正式发布Kimi-K2-Thinking模型后，昇腾与趋境科技开源项目KTransformers通过深度适配，已实现对该模型的高效推理。

**全面生态合作**：2026年6月5日，华为云联合智谱、DeepSeek、Minimax、Kimi、阶跃星辰、百度、美团LongCat、讯飞星火、爱诗科技、生数科技等**20余家TOP模型厂商**，共同发布"**百模千态，云聚共赢**"生态合作计划。

### 3.4 英伟达的公开承认与昇腾算力的全面崛起

2026年5月，英伟达CEO黄仁勋对媒体坦言：

> "很大程度上，我们已经把中国AI芯片市场，让给华为了。"

**昇腾910C 市场表现：**

| 指标 | 数值 |
|------|------|
| 2025年全年出货 | 81.2万颗 |
| 市占率 | 近50% |
| 2026年Q1出货 | 15万颗 |
| 服务头部大模型企业 | 37家 |
| 昇腾384超节点部署 | 超500套 |

清华大学黄小猛课题组的研究验证：迁移优化后的模型在昇腾910B平台上训练速度与英伟达A100相当，差异小于5%；在保持高性能的同时，昇腾平台训练能耗降低约10%-15%。

## 第四章 开源纪元：openPangu的诞生与演进（2025—2026）

### 4.1 2025年8月：更名"openPangu"与大规模开源

2025年8月，华为完成了一次关键的战略调整：将所有盘古开源仓统一更名为"**openPangu**"。同月，华为一口气开源了**1B、7B和718B**共三款模型，其中718B MoE模型是华为首个千亿级MoE模型，总参数量达到7180亿。

轮值董事长徐直军同月正式宣布**CANN华为AI计算架构全面开源开放**，为整个产业界释放了华为从芯片层到模型层的全部技术积累。

### 4.2 2026年3月：全尺寸开源预热

2026年3月，华为中国合作伙伴大会上，华为云CEO周跃峰宣布，华为云将"**全面开放自研盘古大模型**"，推出从1B到718B的全尺寸开源版本矩阵。

核心构建三大能力：
1. 开源开放的"智能体模型池"——开源自研盘古全系列模型，并整合超过160个业界主流模型
2. "后训练"构筑差异化优势——通过注入行业知识和鸿蒙编码能力进行后训练
3. 发布企业级智能体开发平台AgentArts

### 4.3 openPangu 2.0（2026年6月）：从"开源模型"到"开放科学"

**2026年6月12日**，华为开发者大会（HDC 2026）在东莞松山湖召开。余承东正式宣布再度挂帅华为的大模型业务，并发布了全面升级的开源盘古**openPangu 2.0**大模型。

**openPangu 2.0 模型版本：**

| 版本 | 总参数量 | 激活参数量 |
|------|----------|------------|
| 2.0 Pro | 5050亿（505B） | 180亿（18B） |
| 2.0 Flash | 920亿（92B） | 60亿（6B） |

**性能突破：**
- 支持**512K超长上下文窗口**
- 单卡吞吐率达到业内主流开源模型的**2倍**
- 与鸿蒙生态深度融合

**余承东回应质疑：**

> "很多人可能会问，你发布的这个盘古大模型是5050亿参数的，而美国厂家发布的几十T参数的模型，为什么华为没有万亿和几十万亿的大模型？"
>
> "美国企业有几十万卡、上百万卡等更高性能的算力资源。华为把昇腾算力大量支持了国内企业的需要，自己留的卡数量非常有限，还不能满足去训练几十万亿参数的大模型。"
